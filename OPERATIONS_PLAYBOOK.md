## Initial Consultation & Discovery

### Goal
Understand their workflow, identify automation opportunities, build trust

### Pre-Call Preparation (15 minutes)
1. Review their request form submission in admin dashboard
2. Check their company website/LinkedIn
3. Research their tech stack (tools mentioned)
4. Prepare 3 questions specific to their business

### Discovery Call Agenda (30-45 minutes)

**1. Build Rapport (5 mins)**
- "Thanks for reaching out. Tell me about [company] and what you do there."
- Show genuine interest in their business

**2. Understand Current State (15 mins)**
Questions to ask:
- "Walk me through your current process for [task]"
- "What tools are you using today?"
- "How much time does this take per week?"
- "What's the most frustrating part?"
- "What happens when mistakes occur?"
- "Have you tried to automate this before?"

**3. Explore Desired State (10 mins)**
- "Ideally, what would this process look like?"
- "If this worked perfectly, what would change for you?"
- "What does success look like in 3 months?"

**4. Technical Assessment (5 mins)**
- "Do these tools have APIs or Make.com integrations?" (check during call)
- "Who controls access to these accounts?"
- "Any security/compliance requirements?"

**5. Set Expectations (5 mins)**
- "Here's how I typically work: [explain process]"
- "Typical timeline for this would be [X weeks]"
- "Investment would likely be in the range of Â£[low]-Â£[high]"

**6. Close (5 mins)**
- "Any questions for me?"
- "I'll email you a proposal by [specific date/time]"

## Project Delivery

### Week 1-2: Development Phase

**Day 1: Setup**
1. Log into admin dashboard, mark request as "In Progress"
2. Create project folder: `[Client Name] - [Project Name]`
3. Set up Make.com scenario in your testing account
4. Document initial plan

**Daily Updates (Optional but Recommended):**
- Send quick Loom video showing progress
- "Here's what I built today..."
- Builds trust, prevents scope creep

**Development Checklist:**
- [ ] Build main automation flow
- [ ] Add error handling (email you if something fails)
- [ ] Test with sample data
- [ ] Document each step (comments in Make.com)
- [ ] Create fallback/backup scenario if applicable

**Testing Protocol:**
1. Test with sample data (at least 5 scenarios)
2. Test edge cases (missing fields, duplicates, errors)
3. Test error notifications
4. Verify data accuracy
5. Check timing/delays

**Common Make.com Best Practices:**
- Always include error handlers
- Use routers for conditional logic
- Add data validation filters
- Include email notifications for failures
- Log important actions to Google Sheets for audit trail

### Week 2-3: Review & Refinement

**Demo Video Creation:**
1. Record Loom video (10-15 mins)
2. Show entire flow start to finish
3. Explain what happens at each step
4. Show where data goes
5. Demo error handling
6. Explain monitoring

**Video Script Template:**
```
"Hi [Name], here's your [automation] in action.
```
